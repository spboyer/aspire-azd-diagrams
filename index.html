Diagram 3: aspire deploy → azd aspire deploy — Data Flow

Step-by-step data flow when a .NET developer runs aspire deploy.

sequenceDiagram
    participant Dev as 👩‍💻 Developer
    participant Aspire as 🟣 aspire deploy
(microsoft/aspire)
    participant Disk as 💾 Disk
(/tmp/aspire-out/)
    participant AzdExt as 🟢 azd aspire deploy
(extension)
    participant AzdCore as 🔵 azd core
(gRPC host)
    participant Azure as ☁️ Azure

    Dev->>Aspire: aspire deploy --target azure

    Note over Aspire: Step 1: Publish Pipeline
    Aspire->>Aspire: publish-prereq:
Generate manifest.json
    Aspire->>Aspire: publish:
Generate Bicep files
(Azure.Provisioning SDK)

    Note over Aspire,Disk: Step 2: Write Artifacts
    Aspire->>Disk: manifest.json
    Aspire->>Disk: main.bicep
    Aspire->>Disk: *.module.bicep
    Aspire->>Disk: *-roles.module.bicep

    Note over Aspire,AzdExt: Step 3: Invoke azd
    Aspire->>AzdExt: Process.Start("azd",
"aspire deploy
--from-manifest /tmp/aspire-out/
--output json")

    Note over AzdExt,AzdCore: Step 4: Extension Work
    AzdExt->>Disk: Read manifest.json
    AzdExt->>Disk: Read *.bicep files
    AzdExt->>AzdCore: gRPC: Get environment
    AzdCore-->>AzdExt: Environment config
    AzdExt->>AzdCore: gRPC: Get subscription
    AzdCore-->>AzdExt: Azure subscription
    AzdExt->>Azure: Provision resources (Bicep deploy)
    Azure-->>AzdExt: Resource outputs
    AzdExt->>Azure: Build + push containers to ACR
    Azure-->>AzdExt: Image digests
    AzdExt->>Azure: Deploy to Container Apps
    Azure-->>AzdExt: Deployment complete

    Note over AzdExt,Aspire: Step 5: Return Results
    AzdExt-->>Aspire: JSON: {status, resourceGroup,
endpoints, dashboardUrl}

    Note over Aspire,Dev: Step 6: Display Summary
    Aspire-->>Dev: ☁️ Target: Azure
📦 rg-myapp-dev
🌐 https://api.myapp.dev
📊 Dashboard: https://...

Diagram 6: Publish Mode Comparison — fullAzd vs fullAspire

The extension must handle both paths. fullAspire is the future direction.

flowchart TB
    subgraph MODE_A["fullAzd Mode (v0 resources — legacy)"]
        direction LR
        A_AH["AppHost"] -->|"dotnet run
--publisher manifest"| A_MF["manifest.json
(v0 resources,
NO deployment blocks)"]
        A_MF -->|"Extension reads"| A_EXT["azd aspire extension
GENERATES Bicep
(Go templates)"]
        A_EXT -->|"main.bicep
resources.bicep
*.tmpl.yaml"| A_DEPLOY["Provision
+ Deploy"]
    end

    subgraph MODE_B["fullAspire Mode (v1 resources — modern, FUTURE)"]
        direction LR
        B_AH["AppHost"] -->|"dotnet run
--publisher manifest"| B_MF["manifest.json
(v1 resources,
WITH deployment blocks)
+ *.module.bicep files"]
        B_MF -->|"Extension reads"| B_EXT["azd aspire extension
CONSUMES Bicep
(Aspire generated)"]
        B_EXT -->|"Uses Aspire's
main.bicep
*.module.bicep"| B_DEPLOY["Provision
+ Deploy"]
    end

    subgraph KEY["🔑 Key Difference"]
        direction LR
        K1["fullAzd: Extension GENERATES infrastructure"]
        K2["fullAspire: Extension CONSUMES infrastructure"]
        K3["Extension must support BOTH for backward compat"]
    end
